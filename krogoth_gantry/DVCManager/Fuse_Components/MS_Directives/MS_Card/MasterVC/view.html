Cards

Description:

ms-card directive can be used for creating cards that follow Google's Material Design specifications. Currently it has 10 different pre-defined templates but you can always create your own templates and use them with the directive.

Attribute Description
template This attribute must be present in order to ms-card directive to work. It works similar to ng-include directive. You can either provide the path of the html template file or the variable that holds the path of the html template file.
ng-model Directive uses ng-model to hold the card data. Data passed with using ng-model will be available in the template files within variable named card.
Example:

Above directive will use my-custom-template.html file as a template. And you will be able to access to the vm.cardModel object through card variable inside the template.

Consider the following object: vm.cardModel = { title : 'My Card', description: 'My card description' }

It will be available in the template file as: card = { title : 'My Card', description: 'My card description' }

So you will be able to access the title by doing {{ card.title }} inside the template file.

Cards are useful for showing excerpts, quick information or notifications. Using ms-card directive you can easily create different cards compatible with Google's Material Design specifications.

ms-card directive templates have pre-defined content areas. The reason for that is to force you to create cards that follows both Google's Material Design specifications and Fuse Theme's general design rules. But of course you can create your own designs and templates and use them with ms-card directive.
